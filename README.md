# Zenith Documentation

The canonical source of truth for Zenith documentation, changelogs, and release alignment.

## Structure

- **`changelog/`**: Versioned changelogs for Zenith releases.
- **`frontmatter.md`**: Specification for page metadata.
- **`VERSIONING.md`**: Conceptual overview of version visibility.
- **Core Guides**: Documentation files moved from the local core repository.

## Frontmatter Convention

All Markdown files should include a minimal YAML frontmatter:

```yaml
---
title: "Page Title"
order: 1
version: "0.4.0"
---
```
