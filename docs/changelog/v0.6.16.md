# zenith-docs v0.6.16

## Highlights

- Zenith imported the canonical `@zenithbuild/language` and `@zenithbuild/language-server` packages into the monorepo and shipped the first editor-tooling train as part of the framework release line.
- Docs and release policy now describe the `beta` / `train` / `master` release model and the tag-driven publish flow.

## Language / Language Server / Editor Tooling

- VS Code-compatible editor support now ships through the canonical Zenith grammar and language server instead of ad hoc editor rules.
- Canonical TypeScript tokenization, compiler-backed diagnostics, hover docs, and completion coverage landed for `.zen` authoring.
- CI coverage was added for grammar embedding, hover docs, completions, and `ZEN-DOM-*` diagnostic behavior.

## Verification / Stability

- Framework publishing moved to the tag-driven `CI -> npm publish -> GitHub Release` workflow with OIDC-only normal publishing.
- Docs now track the release model as part of the canonical framework contract instead of keeping editor-tooling release behavior implicit.
