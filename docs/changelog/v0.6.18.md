# zenith-docs v0.6.18

## Highlights

- Zenith locked the recent framework, routing, and editor-tooling work into one coherent release line.
- The release focused on contract-correct fixes with regression coverage instead of site workarounds or split package/version stories.

## Framework / Compiler / CLI / Runtime

- Fixed slot-scope owner attribution in the CLI component-occurrence path so slotted refs, state, and signals retain parent scope through component expansion and emitted child props.
- Fixed marker namespace collisions between ref markers and dynamic attr markers by separating ref markers from `data-zx-*` attr markers on the same node.
- Fixed runtime SVG `class` binding application so SVG elements bind through SVG-safe attribute writes while preserving existing HTML `class` behavior.
- Fixed CLI post-compiler expression rewriting so mixed component-local expressions keep rewritten reactive identifiers and rewritten local const symbols together in final built binding functions.

## Routing / Navigation / Site

- Server routing is now the authority for route precedence, guard/load ordering, redirect/deny behavior, 404/error handling, and manifest parity.
- Client routing now mirrors server truth through explicit opt-in links, fetch-before-commit data refresh, deterministic history/scroll/focus rules, and deterministic lifecycle hooks.
- The current site transition prototype stays narrow: `/`, `/about`, `/blog`, and `/docs` route-entry only. Docs deep-hash links remain outside soft navigation.
- The site dev/build wrapper now uses the current workspace-linked toolchain path and survives rebuild handoffs without poisoning the live dev state.

## Language / Language Server / Editor Tooling

- The compiler JSON bridge now emits structured `diagnostics[]` alongside legacy `warnings[]`, preserving additive compatibility while exposing compiler-known errors and warnings with stable codes, ranges, severities, and metadata.
- The language server now prefers structured compiler diagnostics when available and falls back to legacy `warnings[]` only for transition compatibility.
- First compiler-code-backed quick fixes shipped for `ZEN-DOM-QUERY`, `ZEN-DOM-LISTENER`, and `ZEN-DOM-WRAPPER`.
- Hover and completion coverage expanded from canonical docs for Zenith primitives and documented `on:*` events, including `on:esc`, `on:hoverin`, `on:hoverout`, `zenOn`, `zenResize`, `zenWindow`, and `zenDocument`.
- Startup and packaging were hardened so the bundled VS Code-compatible extension launches the packaged server over `--stdio`, and the direct Node LSP entry boots cleanly.

## Verification / Stability

- Added focused regression coverage for slot-scope ownership, marker namespace separation, SVG `class` binding, component-expression rewriting, and final props-prelude ref emission.
- Packaged VS Code verification passed for compiler-backed diagnostics, `strictDomLints`, supported `ZEN-DOM-*` quick fixes, canonical hovers, and canonical completions.
- Direct Neovim LSP verification passed on the default Node `--stdio` path for the same diagnostic, hover, completion, and quick-fix surfaces.
- The verified site path no longer needed local site workarounds for the framework failures after the fixes landed.
