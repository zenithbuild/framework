# zenith-docs v0.6.18

## Highlights

- Zenith locked a set of compiler, CLI, and runtime regressions that had surfaced in real site hydration and component-expansion flows.
- The release focused on contract-correct fixes with regression coverage instead of site workarounds.

## Core / Compiler / CLI / Runtime

- Fixed slot-scope owner attribution in the CLI component-occurrence path so slotted refs, state, and signals retain parent scope through component expansion and emitted child props.
- Fixed marker namespace collisions between ref markers and dynamic attr markers by separating ref markers from `data-zx-*` attr markers on the same node.
- Fixed runtime SVG `class` binding application so SVG elements bind through SVG-safe attribute writes while preserving existing HTML `class` behavior.
- Fixed CLI post-compiler expression rewriting so mixed component-local expressions keep rewritten reactive identifiers and rewritten local const symbols together in final built binding functions.

## Verification / Stability

- Added focused regression coverage for slot-scope ownership, marker namespace separation, SVG `class` binding, component-expression rewriting, and final props-prelude ref emission.
- The verified site path no longer needed local site workarounds for these core failures after the framework fixes landed.
