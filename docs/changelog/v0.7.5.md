# zenith-docs v0.7.5

Latest stable release for the Zenith framework train.

## Highlights

- Zenith now ships a rewrite-free `static-export` target, canonical presence/navigation-shell/overlay guidance, truthful route-owned cookie sessions, explicit resource routes, and one router-owned freshness bridge on top of the existing full-stack contract.
- Hosted `vercel` and `netlify` targets now match the shipped local server surface for page routes, page-route cookie sessions, hosted resource `json(...)` / `text(...)`, and the existing `/_zenith/image` runtime endpoint.
- Final build/static HTML image materialization now belongs to bundler-owned truth while runtime/server image materialization stays explicitly separated.

## ✨ New Features & Improvements

- `target: "static-export"` now emits rewrite-free concrete public files from canonical route truth, with explicit `exportPaths` for dynamic prerender routes and base-path-aware output/preview behavior.
- `@zenithbuild/runtime` and `@zenithbuild/router` now ship `zenPresence(...)`, the optional `presence(...)` alias, `zenNavigationShell(...)`, and the canonical always-mounted overlay/sheet docs/demo pattern.
- `@zenithbuild/cli` and the server route contract now support dedicated resource routes with `json(...)`, `text(...)`, and attachment-style `download(...)` on the existing `guard(ctx)` / `action(ctx)` / `load(ctx)` model.

## 🧱 Runtime / Server Contract Improvements

- Route-owned multipart uploads now work through native `await ctx.request.formData()` in dev, preview, and packaged node without inventing a second mutation surface.
- `ctx.auth` is now truthful in dev, preview, and packaged node with cookie-backed `getSession()`, `requireSession(...)`, `signIn(...)`, and `signOut()`.
- `refreshCurrentRoute()` now gives page apps one explicit freshness bridge after resource writes or resource-route auth changes while keeping page HTML flows automatically fresh and resource routes explicit/manual.
- Final build/static HTML image materialization now happens from bundler-owned artifact truth instead of CLI post-build mutation, while runtime/server image behavior stays intentionally separate.

## 🌐 Hosted Deployment Parity

- Hosted `vercel` and `netlify` page routes now run on the packaged shared runtime with proven cookie-session redirect and `Set-Cookie` parity.
- Hosted resource routes now support `json(...)`, `text(...)`, `redirect(...)`, and `deny(...)`, including auth and staged-cookie behavior on those supported responses.
- Hosted `/_zenith/image` now preserves packaged-node semantics for `400 missing_url`, successful binary image bodies, `Content-Type`, and `Cache-Control`.

## 📦 Published Versions

- Lockstep train `0.7.5` covers `@zenithbuild/bundler`, `@zenithbuild/cli`, `@zenithbuild/compiler`, `@zenithbuild/core`, `@zenithbuild/router`, `@zenithbuild/runtime`, `@zenithbuild/language`, and `@zenithbuild/language-server`.
- Optional platform packages for `@zenithbuild/bundler-*` and `@zenithbuild/compiler-*` move with the same `0.7.5` train.
