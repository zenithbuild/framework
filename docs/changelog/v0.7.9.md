# zenith-docs v0.7.9

Stable follow-up patch for the compiler foreign-syntax hardening release.

## Highlights

- `v0.7.8` introduced the correct compiler-owned foreign-syntax gate, but the first cut was too broad: it also rejected legitimate component handler props like `onClick` and `onPress`.
- Zenith now keeps the foreign-syntax rejection exactly where it belongs: native DOM/template syntax from other frameworks still fails fast, while component props that transport handler functions across Zenith components remain valid.
- This patch is a framework/compiler correction only. It does not broaden syntax support or change runtime behavior.

## Compiler Boundary Correction

- The foreign-syntax scanner now distinguishes native markup from component tags before classifying event-like attributes.
- Native foreign event props such as `onClick`, `onclick`, and directive forms like `@click` remain compiler errors on real DOM elements.
- Component tags may continue to accept handler-shaped props such as `onClick`, `onPress`, `onInput`, and `onSubmit` when those values are passed through as ordinary component props and bound canonically inside the child with `on:<event>={handler}`.

## Diagnostics and Contract Stability

- Added regression coverage to lock the component-prop boundary alongside the existing foreign-syntax diagnostic snapshots.
- The structured diagnostic contract added in `v0.7.8` remains unchanged for actual foreign syntax violations.

## What This Release Does Not Change

- No new event or control syntax was added.
- No runtime semantics changed.
- No compatibility layer for foreign frameworks was introduced.
- Native foreign template/event syntax in `.zen` is still rejected at compile time.

## Published Versions

- Lockstep train `0.7.9` covers `@zenithbuild/bundler`, `@zenithbuild/cli`, `@zenithbuild/compiler`, `@zenithbuild/core`, `@zenithbuild/router`, `@zenithbuild/runtime`, `@zenithbuild/language`, and `@zenithbuild/language-server`.
- Optional platform packages for `@zenithbuild/bundler-*` and `@zenithbuild/compiler-*` move with the same `0.7.9` train.
