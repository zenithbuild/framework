# zenith-docs v0.8.0

Framework maintenance, adapter-boundary planning, security cleanup, and release-train hygiene for the Zenith lockstep package train.

## Highlights

- The internal adapter contract is confirmed around `AdapterDriver` and `resolveBuildAdapter(...)`, while the public adapter plugin API remains unshipped.
- Built-in Vercel and Netlify adapters are verified behind the normalized internal adapter path and shared hosted infrastructure.
- The delegated adapter surface RFC records a future direction for adapter factories without approving implementation.
- Global middleware V1 remains narrow: TypeScript root middleware only, `next` / `redirect` / `deny` only, and no arrays, nested scopes, controlled headers, or route-check participation.
- The release train now includes `@zenithbuild/extension-registry` alongside the framework packages that depend on it.
- Oversized tests, docs, scripts, and bundler modules were split into focused files under the file-size policy.

## Adapter And Plugin Boundary

- `target` remains the stable public deployment selector.
- The raw `adapter` object remains an advanced deployment surface, not a broad public plugin API.
- Delegated adapter factories such as future `vercel()` / `netlify()` helpers remain RFC material only.
- Adapter packages cannot register middleware, mutate route security semantics, or install compiler, bundler, runtime, router, or route-check hooks.

## Middleware Planning

- Post-V1 middleware expansion was evaluated and intentionally deferred or rejected where appropriate.
- TypeScript-only root middleware remains the shipped contract.
- JavaScript-authored middleware remains unsupported until a separate language-mode contract is approved.
- Route-check remains guard-only and advisory; it does not execute global middleware.

## Security And Tooling

- Legacy compiler `ws` surface was removed and the workspace pins a patched `ws` version.
- Editor/tooling diagnostics closeout work landed for the framework language-server path.
- esbuild was updated for GHSA-gv7w-rqvm-qjhr.
- Benchmark-only Astro, Vite, Nuxt, and Next dependencies were removed with the benchmark/performance surface after high-severity Dependabot alerts.
- No active `astro` manifest or lockfile references remain, and `bun audit --audit-level high` passes on the post-removal dependency surface.
- Route data contract edge cases and framework reliability paths received additional coverage.

## Maintainability

- CLI drift-gate, dev-server, and server-routing tests were split into focused suites.
- Runtime dom-binding and integration suites were split while preserving behavior.
- Bundler helper clusters and the main output phase were split into smaller modules.
- Docs/site generators, AI endpoint generation, overlay sheet docs, and public-asset sync helpers were decomposed into narrower files.
- File-size audit policy was calibrated for generated and legacy snapshots.

## What This Release Does Not Change

- No public adapter plugin API is shipped.
- No middleware arrays, nested scopes, controlled headers, `middleware.js`, or route-check middleware participation are shipped.
- No Graphify workflow is required.
- No native binary distribution migration is introduced.

## Published Versions

- Lockstep train `0.8.0` covers `@zenithbuild/bundler`, `@zenithbuild/cli`, `@zenithbuild/compiler`, `@zenithbuild/core`, `@zenithbuild/extension-registry`, `@zenithbuild/router`, and `@zenithbuild/runtime`.
- Optional platform packages for `@zenithbuild/bundler-*` and `@zenithbuild/compiler-*` move with the same `0.8.0` train.
- Standalone editor packages remain separately release-owned.
