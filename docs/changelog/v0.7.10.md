# zenith-docs v0.7.10

Stable package-surface patch for the Zenith framework release train.

## Highlights

- The router source already imported `template-refresh.js` from `template.js`, but the published `@zenithbuild/router@0.7.9` tarball did not include that helper file.
- Fresh consumer installs could therefore fail during framework builds with `Cannot find module ... @zenithbuild/router/template-refresh.js`.
- Zenith now publishes the router refresh helper as part of the lockstep framework package set, so hosted builds resolve the same package surface that local workspace builds use.

## Router Package Surface

- `@zenithbuild/router` now includes `template-refresh.js` in its package `files` allowlist.
- The router template contract test now asserts both that the helper exists in source and that the package manifest keeps it in the publish surface.
- This protects the template bundle from drifting between source checkouts and npm tarballs.

## Consumer Build Compatibility

- This release is targeted at hosted/fresh-install consumers, including Vercel builds that install `@zenithbuild/*` from npm.
- The fix belongs in the framework monorepo package train rather than in an individual site, because the missing file is part of the router package contract.
- Sites should upgrade their Zenith dependencies to `0.7.10` instead of patching installed packages.

## What This Release Does Not Change

- No new router API was added.
- No route behavior changed.
- No runtime workaround was introduced.
- No site content, route, or deployment configuration is required to compensate for the missing file.

## Published Versions

- Lockstep train `0.7.10` covers `@zenithbuild/bundler`, `@zenithbuild/cli`, `@zenithbuild/compiler`, `@zenithbuild/core`, `@zenithbuild/router`, `@zenithbuild/runtime`, `@zenithbuild/language`, and `@zenithbuild/language-server`.
- Optional platform packages for `@zenithbuild/bundler-*` and `@zenithbuild/compiler-*` move with the same `0.7.10` train.
