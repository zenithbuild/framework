# zenith-docs v0.7.0

## Highlights

- Zenith now ships native `Image` v1 support with secure remote-image guardrails and static local-image materialization.
- The public site moved onto a stronger Directus-backed editorial surface system across docs, blog, changelog, and about.
- Benchmark work now captures rebuild behavior and publishes clearer performance evidence without overstating the current rebuild story.

## Framework / Build / Image Pipeline

- Added framework-native `Image` v1 support as the canonical image surface.
- Added image config validation and build/dev/preview image pipeline wiring across the framework toolchain.
- Materialized local images statically and kept remote image handling behind explicit allowlists and guardrails.
- Hardened typed embedded markup rewriting, hoisted expression handling, and narrower page-only rebuild behavior in the CLI/build pipeline.

## Site / Docs / Editorial Surfaces

- Rebuilt the public site around stronger editorial landing/detail surfaces for About, Docs, Blog, and Changelog.
- Moved the site onto Directus-backed blog and changelog source truth where intended, including live dev rendering fixes for real CMS content.
- Tightened docs landing/detail composition, navigation, and rendering so docs stay utility-first while matching the broader Zenith design family.

## Benchmarks / Evidence

- Added benchmark fixtures, rebuild-aware measurement flow, and generated performance result docs for the current comparison set.
- Documented rebuild caveats honestly so current benchmark claims stay evidence-first and scoped to the trusted result path.

## CMS / Workflow

- Added the Directus editorial workspace updates, extension build tooling, and repo-sync flows needed for the CMS-side publishing workflow.

## Verification / Release Readiness

- Framework integration passed the repo CI gate.
- Site, benchmarks/docs, and CMS batches all passed their branch-specific verification gates before the train release prep.
- The release train remains lockstep at `0.7.0` across the published framework packages and their internal references.
