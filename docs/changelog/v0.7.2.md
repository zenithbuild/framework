# zenith-docs v0.7.2

## Highlights

- Zenith now has a complete deployment target matrix across `static`, `vercel-static`, `netlify-static`, `vercel`, `netlify`, and `node`.
- The deployment contract is now hardened with canonical `.zenith-output` manifests, packaged server output, and non-root `basePath` support across build, preview, adapters, router behavior, and SSR.

## Deployment Model

- `zenith build` now writes canonical intermediate output before adapter packaging, with route classification fixed upstream in manifest and server-package layers.
- Server-capable targets now consume `.zenith-output/server/` as the packaged runtime boundary instead of inferring route meaning inside host adapters.
- `zenith preview` is target-aware and honors the same deployment contract as the built target.

## Base Path / Public URL Contract

- `basePath` is now the single public mount-path knob for framework deploys.
- Public route URLs, asset URLs, router endpoints, `/_zenith/image`, and `/__zenith/route-check` all follow `basePath`.
- Canonical route identity remains base-path free in manifests and route classification so adapters only map public URLs onto already-classified output.

## Verification / Release Readiness

- Deployment smoke coverage now runs as an explicit CI contract.
- Manifest and server-output fixture snapshots are in place to catch schema drift early.
- This stable cut moves the lockstep framework packages and internal references to `0.7.2`.
