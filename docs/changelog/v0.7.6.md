# zenith-docs v0.7.6

Stable release for the Zenith framework train.

## Highlights

- Resource-route streaming and SSE are now first-class standalone server-contract results (`stream(...)`, `sse(...)`) with local + hosted parity.
- Zenith now supports `src/api/**` as resource-route DX aliasing and explicit route-local middleware composition through `withMiddleware(...)`.
- Benchmark publication is hardened and truthful by policy: Zenith determinism remains a hard gate, while external framework determinism is reported as caveat metadata.
- The optimization lane materially reduced interactive emitted payload while preserving determinism, zero-JS static behavior, and hosted/resource parity.
- Structural decomposition reduced hot-path file risk in bundler, compiler, and CLI with scoped file-size CI enforcement.

## ✨ New Features & Improvements

- Added resource-route streaming/SSE helper support on the existing server contract (`stream(...)`, `sse(...)`) and kept route-kind enforcement explicit.
- Added `src/api/**` discovery sugar for `src`-layout projects while compiling to the existing resource-route model.
- Added explicit route-level middleware composition (`withMiddleware(...)`) for `guard`, `load`, `action`, and resource handlers without introducing global pipeline semantics.

## 📊 Performance and Benchmark Integrity

- Publication benchmark harness now fails on empty outputs, timeouts treated as success, and hydration/runtime failures.
- Determinism publication policy is explicit and enforced in reporting artifacts.
- Interactive emitted payload was reduced across runtime/router/page production assets with deterministic output constraints preserved.

## 🧱 Structural Maintainability

- Completed Wave 1 (bundler), Wave 2 (compiler), and Wave 3 (CLI hot-path) decomposition work under Phase 21.
- Added file-size governance and scoped CI enforcement for high-risk paths.
- Kept decomposition behavior-preserving with determinism/publication/parity gates intact.

## 📦 Published Versions

- Lockstep train `0.7.6` covers `@zenithbuild/bundler`, `@zenithbuild/cli`, `@zenithbuild/compiler`, `@zenithbuild/core`, `@zenithbuild/router`, `@zenithbuild/runtime`, `@zenithbuild/language`, and `@zenithbuild/language-server`.
- Optional platform packages for `@zenithbuild/bundler-*` and `@zenithbuild/compiler-*` move with the same `0.7.6` train.
