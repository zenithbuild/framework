# zenith-docs v0.7.1

## Highlights

- Zenith's framework release path now uses one direct trusted-publishing contract across `beta`, `train`, and `latest`.
- Stable publishes now happen directly from `master` under plain semver tags, instead of relying on post-publish latest promotion.

## Publishing Contract

- `vX.Y.Z-beta.N` tags publish to npm dist-tag `beta` from `beta`.
- `vX.Y.Z-train.N` and `vX.Y.Z-rc.N` tags publish to npm dist-tag `train` from `train`.
- `vX.Y.Z` tags publish to npm dist-tag `latest` from `master`.
- Stable releases are distinct semver publishes, not republished train versions.

## Workflow / Repo Hygiene

- Kept the framework publish workflow on the real `publish-packages.sh` entrypoint.
- Removed the obsolete latest-promotion workflow and helper scripts.
- Renamed the remaining publish workflow labels and outputs so they describe generic release channels instead of only the train line.

## Verification / Release Readiness

- The trusted-publishing workflow, release metadata flow, and internal release-policy docs now agree on the same branch and tag contract.
- This stable cut moves the lockstep framework packages and their internal references to `0.7.1`.
