# zenith-docs v0.7.8

Stable framework hardening release for the Zenith train.

## Highlights

- Guard exposed a real drift path where foreign framework syntax could leak into `.zen` source and survive long enough to appear in output.
- Zenith now blocks that class of issue in the compiler itself, before parse/emission, with explicit diagnostics and canonical rewrite hints.
- The release also locks the diagnostic contract with golden tests and documents the invalid-vs-canonical forms in the public docs.

## Compiler Hardening

- Added a compiler-owned pre-parse invariant that rejects foreign control/template syntax in `.zen` files.
- Covered Blade/Twig-style directives (`@if`, `@else`, `@elseif`), Svelte blocks (`{#if}`, `{/if}`, `{#each}`, `{/each}`), Vue directives (`v-if`, `v-else`, `v-for`), and parseable React conditional markup artifacts.
- This keeps foreign framework syntax from reaching emitted output or runtime surfaces.

## Event Syntax Enforcement

- Added explicit compiler rejection for foreign event syntax such as `@click`, `@input`, `@change`, `onClick`, `onInput`, and `onclick`.
- Diagnostics point authors back to the canonical Zenith event contract: `on:<event>={handler}`.
- This release preserves the existing Zenith event model; it does not introduce compatibility aliases for foreign frameworks.

## Diagnostics and Author Guidance

- Added structured diagnostics with stable codes for foreign control syntax and foreign event syntax.
- Locked the wording and hint shape with golden compiler tests so the invariant remains stable over time.
- Added a short public docs note showing the invalid foreign tokens and the canonical Zenith forms.

## What This Release Does Not Change

- No runtime semantics changed.
- No new template syntax was added.
- No UI-layer cleanup or workaround was added.
- This is a framework/compiler enforcement release, not a Guard product release.

## Published Versions

- Lockstep train `0.7.8` covers `@zenithbuild/bundler`, `@zenithbuild/cli`, `@zenithbuild/compiler`, `@zenithbuild/core`, `@zenithbuild/router`, `@zenithbuild/runtime`, `@zenithbuild/language`, and `@zenithbuild/language-server`.
- Optional platform packages for `@zenithbuild/bundler-*` and `@zenithbuild/compiler-*` move with the same `0.7.8` train.
