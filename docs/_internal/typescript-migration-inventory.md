## Phase-1 constraints

- Public entrypoints, module format, and shipped JS output paths stay unchanged.
- Core packages are TS-authored but still ship JS in `dist/` with declarations alongside the emitted JS.
- `allowJs + checkJs` remains intentionally scoped for `runtime` and `cli` to keep the migration incremental without rewriting large orchestration modules.
