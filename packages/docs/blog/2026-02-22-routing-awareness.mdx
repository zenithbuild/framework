# Routing Awareness Fix

Zenith now enforces server-first routing invariants across dev and preview:

- Server route matching uses manifest data for every request.
- Default links retain hard reload behavior.
- Soft navigation remains explicit and marker-based.

## Why this matters

Route correctness no longer depends on runtime assumptions. A direct URL visit and an in-app link resolve the same route and parameters.

## What to verify

- Route parity for static, dynamic, and catch-all URLs.
- No runtime URL mutation primitives in framework outputs.
- Inline SSR payload present exactly once.
